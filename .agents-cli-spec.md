# Agent Spec: Party Store Supply Chain Agent

## Overview
The Party Store Supply Chain Agent is designed to optimize supply chain operations for a party store. It uses historical sales data (orders) and shipment data in BigQuery to forecast demand, manage inventory, and suggest reordering decisions. It utilizes A2UI to display dashboards and charts, and A2A to communicate with a Procurement Agent.

## Example Use Cases
1.  **Demand Forecasting:** User asks "What is the sales forecast for Halloween decorations?". The agent queries BigQuery, calculates a forecast (considering seasonality), and returns a line chart showing historical and forecasted sales.
2.  **Inventory Health Check:** User asks "How is our inventory looking?". The agent checks current stock levels (calculated from shipments received minus orders), compares them with forecasted demand, and displays a dashboard with stock levels (bar chart) and reorder alerts.
3.  **Automatic Reordering (A2A):** The agent detects that inventory for a product is below the reorder point. It automatically triggers a request to the `Procurement Agent` to create a purchase order, and informs the user.

## Data Foundation: BigQuery Dataset
The agent connects to a BigQuery dataset with the following tables:
*   `orders`: `order_id` (STRING), `product_id` (STRING), `order_date` (DATE), `quantity` (INTEGER), `customer_id` (STRING)
*   `shipments`: `shipment_id` (STRING), `product_id` (STRING), `ship_date` (DATE), `receive_date` (DATE), `quantity` (INTEGER)

## Tools Required
1.  **BigQuery Tool:** To run queries on the `orders` and `shipments` tables.
2.  **Forecasting Tool:** Python logic (or a model) to calculate forecasts from historical data.
3.  **A2UI Renderer:** To generate A2UI JSON payloads for dashboards and charts.
4.  **Procurement Client (A2A):** To send reorder requests to the Procurement Agent.

## Constraints & Safety Rules
*   Do not modify BigQuery data directly (read-only access to orders/shipments, though it might need to log recommendations somewhere if required, but keep it read-only for core data).
*   Only recommend reorders, do not execute them without user confirmation (or A2A handoff for review).

## Success Criteria
*   Successfully queries BigQuery to get historical data.
*   Generates a reasonable forecast showing seasonal spikes (e.g., Halloween).
*   Correctly calculates current inventory.
*   Generates valid A2UI JSON for charts and tables.
*   Simulates A2A call to Procurement Agent.
